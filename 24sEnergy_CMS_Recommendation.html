แนะนำใช้ Sanity — เหตุผล

5 เหตุผลที่ Sanity เหมาะกับ 24sEnergy มากที่สุด

🎨

1. Studio UI ปรับแต่งได้

หน้า admin สวยกว่าทุกตัว ทีมที่ไม่ใช่ dev แก้คอนเทนต์ได้ทันที พร้อม real-time preview เห็นผลลัพธ์ทันทีบนเว็บจริง

🌐

2. Bilingual ตั้งง่าย

Plugin @sanity/document-internationalization แค่ install ก็พร้อมใช้ ทุก field มี TH + EN และ admin มีปุ่มสลับภาษา

🖼️

3. Image CDN ระดับโลก

รูปอัพโหลดแล้วได้ CDN URL ที่ resize/crop/optimize ได้ผ่าน URL params · WebP/AVIF อัตโนมัติ ช่วยให้เว็บโหลดเร็ว

⚡

4. Performance ดีเยี่ยม

Static generation ผ่าน Next.js + Sanity CDN = LCP < 2s · ดีกว่า WordPress ที่ render ทุกครั้งที่มี request

💸

5. Free Tier ใหญ่

3 admin users + 10,000 documents + 500k API calls/เดือน · พอใช้ 2 ปีแรกแน่นอน · จ่ายเฉพาะตอนโตจริงๆ

🛡️

6. Backup & Security ฟรี

ทีม Sanity ดูแลให้ทั้งหมด — daily backup, DDoS protection, 99.99% uptime SLA · ไม่ต้องมี DevOps

ลูกค้า Sanity ระดับโลก

Figma, Netflix, Nike, Sonos, Loom, Eurostar ใช้ Sanity จัดการเว็บไซต์หลัก · เป็นเครื่องยืนยันว่า scale ขึ้นได้แน่นอน

ระบบ 2 ภาษา (TH/EN)

ตั้งค่าใน Sanity แล้วจะได้ admin ที่มี toggle ภาษาทุก field

// sanity.config.ts
import {documentInternationalization} from '@sanity/document-internationalization'

export default defineConfig({
  plugins: [
    documentInternationalization({
      supportedLanguages: [
        {id: 'th', title: 'ไทย'},
        {id: 'en', title: 'English'}
      ],
      schemaTypes: ['product', 'project', 'solution', 'blogPost']
    })
  ]
})

หลังจากนี้ทุก document จะมี:

  • ปุ่มสลับภาษาบน toolbar — กดเดียวสลับ TH ↔ EN
  • Document ตัวเดียวกันแต่คนละภาษา link กันอัตโนมัติ
  • Admin จะเตือนถ้ายังไม่ได้แปลภาษาใด
  • API เรียกได้ตามภาษาผ่าน param ?lang=th

การจัดการรูปภาพ

Sanity Image CDN — อัพโหลดครั้งเดียวใช้ได้ทุก resolution

วิธีใช้งานในเว็บ

// ภาพหน้า Home Hero ที่ resize
const imageUrl = builder.image(hero.image)
  .width(1600)
  .height(900)
  .fit('crop')
  .format('webp')
  .quality(85)
  .url()

Features ที่ได้ฟรี

  • WebP / AVIF อัตโนมัติ
  • Lazy loading + responsive srcset
  • Hotspot & crop — focus จุดสำคัญ
  • Blur placeholder (LQIP)
  • Global CDN (Cloudflare)
  • EXIF auto-remove (privacy)

ขนาดไฟล์ที่แนะนำให้ทีม content อัพโหลด

· Hero / Project hero: 2000×1200 px, JPG/PNG ไม่เกิน 1 MB (Sanity จะ optimize ให้)
· Product images: 1200×800 px ไม่เกิน 500 KB
· Logo / icon: SVG หรือ PNG transparent
· ห้ามอัพ HEIC จาก iPhone ตรงๆ — แปลงเป็น JPG ก่อน

ขั้นตอน Setup

7 ขั้นตอนตั้งแต่ 0 ถึงพร้อมใช้งาน (ประมาณ 1-2 สัปดาห์)

1

สมัครบัญชี Sanity

ไปที่ sanity.io → Sign up ด้วย Google/GitHub → สร้าง project ชื่อ "24senergy" → เลือก Free plan

2

Init Sanity Studio

รัน npm create sanity@latest -- --project YOUR_ID --dataset production → เลือก clean template → ได้ Sanity Studio ที่ http://localhost:3333

3

กำหนด Schema

สร้างไฟล์ใน /schemas ตามที่ออกแบบไว้ — product.ts, project.ts, solution.ts, homePage.ts ฯลฯ · ใช้เวลา 2-3 วัน

4

ติดตั้ง Localization Plugin

npm i @sanity/document-internationalization → ตั้งค่า config สำหรับ TH/EN → restart studio

5

Deploy Studio

npx sanity deploy → ได้ URL 24senergy.sanity.studio · ทีม content เข้าใช้งานได้ทันที

6

เพิ่ม Users + Permissions

เชิญทีม marketing/content ผ่าน email · กำหนด role: Administrator, Editor (แก้ได้), Viewer (อ่านอย่างเดียว)

7

Import Content เริ่มต้น

คัดลอกเนื้อหาจากเว็บปัจจุบันใส่ Sanity ผ่าน UI หรือ sanity dataset import · พร้อมเริ่มทดสอบกับ frontend

เชื่อมต่อกับเว็บที่ออกแบบไว้

เปลี่ยน HTML static เป็น Next.js + Sanity — ดึงเนื้อหาจาก CMS แทน hard-code

// pages/index.tsx — หน้า Home ดึงข้อมูลจาก Sanity
import {client, urlFor} from '@/lib/sanity'

export async function getStaticProps({locale}) {
  const data = await client.fetch(`
    *[_type == "homePage"][0] {
      hero { eyebrow, title, lede, orbValue },
      kpis[] { value, unit, label },
      partners
    }
  `, {locale})

  return { props: { data }, revalidate: 60 }
}

export default function Home({data}) {
  return (
    <section className="hero">
      <h1>{data.hero.title[locale]}</h1>
      <p>{data.hero.lede[locale]}</p>
    </section>
  )
}

Migration Plan: HTML → Next.js + Sanity

1. เก็บ design ที่ทำไว้ (CSS, layout) · 2. แปลงเป็น React components · 3. เปลี่ยน hard-coded text/image เป็น query Sanity · 4. Deploy บน Vercel · 5. เปิด preview mode ให้ทีมเช็คก่อน publish

Workflow ทีมแก้เนื้อหา

ขั้นตอนการแก้คอนเทนต์ในชีวิตประจำวัน

STEP 1 · LOGIN

เข้า 24senergy.sanity.studio → login ด้วย Google → เห็น dashboard

STEP 2 · EDIT

เลือก document ที่ต้องการแก้ — เช่น Product → HinerCab → แก้ราคา/description/ภาพ

STEP 3 · PREVIEW

กดปุ่ม Preview → เปิด tab ใหม่ดูเว็บจริง (ก่อน publish คนนอกยังไม่เห็น)

STEP 4 · PUBLISH

กด Publish → เว็บอัพเดทใน 30-60 วินาที ผ่าน Next.js ISR · ไม่ต้อง deploy เอง

STEP 5 · ROLLBACK

ถ้าผิดพลาด → History tab → กดย้อนกลับ version ก่อนหน้าได้

STEP 6 · COLLABORATE

ทีมหลายคนแก้พร้อมกันได้ — เห็น cursor ของกันและกัน (real-time)

Plan B: ถ้าต้องการ Self-host

ถ้านโยบายบริษัทต้องการเก็บข้อมูลบน server เอง — ใช้ Strapi

เหตุผลที่อาจเลือก Strapi

  • เก็บข้อมูลบน server ตัวเอง (compliance)
  • ไม่อยาก lock-in กับ SaaS
  • มี dev ดูแลอยู่แล้ว
  • ต้องการ customize admin UI ลึกๆ
  • มีลูกค้าต่างประเทศที่กฎหมายไม่อนุญาตข้อมูลออกนอกประเทศ

ค่าใช้จ่าย Strapi

  • Strapi software: ฟรี (open-source)
  • Server (DigitalOcean/AWS): ~฿500-1,500/mo
  • Database (Postgres): ~฿300-700/mo
  • Image storage (S3/R2): ~฿100-300/mo
  • SSL + domain: ~฿300/ปี

รวม: ~฿1,000-2,500/เดือน

สิ่งที่ต้องเตรียมถ้าใช้ Strapi

· ต้องมี dev ดูแล server, ทำ backup ประจำ, update plugin/security patch
· ตั้ง CDN เอง (Cloudflare) สำหรับ image optimization
· ตั้ง monitoring (Uptime Robot, Sentry)
· มี contract กับ devops vendor หรือ in-house engineer

ค่าใช้จ่ายรวม (Sanity)

ประมาณการสำหรับปีแรก — Sanity Free + Next.js บน Vercel

เมื่อต้อง Upgrade (โต)

· เมื่อมี มากกว่า 3 users หรือ 10k docs → Sanity Growth $99/mo (~฿3,500)

· เมื่อต้องการ Form backend (RFQ) → ใช้ Sanity เป็น DB ได้ฟรี หรือ HubSpot Free
